# Accessibility Testing (a11y)

## Playwright + axe-core

```bash
<workspace>/py/venv/bin/pip install axe-playwright-python
```

```python
# tests/ui/portal/test_a11y.py
from axe_playwright_python.sync_playwright import Axe

def test_dashboard_no_critical_a11y_violations(page, login_user, base_url):
    page.goto(f"{base_url}/dashboard")
    axe = Axe()
    results = axe.run(page)

    violations = [v for v in results["violations"]
                  if v["impact"] in ("critical", "serious")]
    assert len(violations) == 0, \
        f"a11y violations: {[v['id'] + ': ' + v['description'] for v in violations]}"

def test_login_form_is_keyboard_navigable(page, base_url):
    page.goto(f"{base_url}/login")
    page.keyboard.press("Tab")
    assert page.locator("#username").evaluate("el => el === document.activeElement")
    page.keyboard.press("Tab")
    assert page.locator("#password").evaluate("el => el === document.activeElement")
```

## What to test

| Check | Priority |
|---|---|
| WCAG 2.0 AA compliance | High — legal requirement in many regions |
| Keyboard navigation (Tab order) | High |
| Screen reader labels (aria-label, alt text) | High |
| Color contrast ratios | Medium |
| Focus indicators visible | Medium |
| Error messages linked to fields | High |

## Recommended approach

- Run axe on every key page in the journey suite
- Set as non-blocking in CI initially — report violations, don't fail
- Move to blocking after fixing critical violations
- Re-run on every UI change that touches layout or form elements
